HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Tags and elements are different!
  • There is an element for paragraph or block of text "p"
  • HTML Elements are case sensitive
  • "!DOCTYPE html" declaration is letting the browser know to expect HTML on the page.
  • ui = unordered lists and ol = ordered lists

CSS

  • There are three ways to style a webpage using CSS: Inline CSS, Internal CSS Style Sheet, and External CSS Style Sheet.
  • When using an external style sheet, we link the file using a "link'' element placed in the HTML's "head"
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • The margin property acts much like a margin in a word processing document. A margin property indicates how much space we want around the outside of an element.
  • The padding property is often confused withmargin, but it is not the same thing. The padding property adds space around the content inside an element. When you increase padding, you are actually making the entire element bigger!
  • CSS Box Model: Each box consists of margins outside the element, and the element's borders, padding, and content.

Git

  • It may seem obvious, but you should always start with "git" for any git commands in terminal
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Once you have added the content you want, there are several steps you must take to get your code merged to main branch
  • Stage your branch by "git add -A"
  • Commit your changes by "git commit -m message notes here"
  • Confirm your current main branch is up to date "git pull origin main"
  • Push your changes " git push origin "
  • Go to Github to merge your branch into main branch
  • Create a pull request; You will see your branch compared with the main one.
  • Merge pull request
  • Update local repository with new merged in feature
  • git checkout main
  • git pull origin main
  • You can find all local branches by "git branch"
  • You can use the terminal within Visual Studio Code

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A semi-colon is only required when you need to separate stateuments on a single line
  • Javascript is case sensitive! myVariable is not the same as myvariable
  • There are a ton of data types in Java Script Ie. String, Number, Boonlean, Array, Object
  • Everything is an object in Javascript!
  • An operator is a mathematical symbol that produces a result based on two values or variables.
  • Conditionals are code structures used to test if an expression returns true or not. (Common example is 'If..else'.
  • A function is simply a set of instructions that tells the computer how to perform a certain task. Functions are a way to package functionality you wish to reuse.
  • You must call a function by name in order to execute it.
  • Events: Real interactivity on a website requires event handlers. These are code structures that listen for activity in the browser, and run code in response.
  • A value is considered falsy if 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
